## 7. CT201 Monitoring / Grafana / Prometheus

（規則/notifier/dashboard 說明見《監控告警.txt》；這裡只放命令）

```sh
# compose / prometheus
pct exec 201 -- bash -lc 'cd /opt/monitoring && docker compose ps'
pct exec 201 -- sed -n '1,220p' /opt/monitoring/prometheus.yml
pct exec 201 -- curl -X POST http://127.0.0.1:9090/-/reload
pct exec 201 -- bash -lc 'curl -s http://127.0.0.1:9090/api/v1/targets | jq -r ".data.activeTargets[] | [.labels.job,.health,.scrapeUrl] | @tsv"'

# 告警相關 container
pct exec 201 -- bash -lc 'cd /opt/monitoring && docker compose ps gotify alertmanager alertmanager-gotify-bridge prometheus'
# 改規則（同檔內容變更只需 reload；★ 若改的是 bind-mount 單檔需 docker restart 重綁 inode）
pct exec 201 -- docker exec prometheus promtool check rules /etc/prometheus/rules/alerts.yml
pct exec 201 -- curl -s -X POST http://127.0.0.1:9090/-/reload
pct exec 201 -- docker exec alertmanager amtool check-config /etc/alertmanager/alertmanager.yml
pct exec 201 -- curl -s -X POST http://127.0.0.1:9093/-/reload
# 目前 firing/pending
pct exec 201 -- bash -lc 'curl -s http://127.0.0.1:9090/api/v1/alerts | jq -r ".data.alerts[] | [.labels.alertname,.state,.labels.severity] | @tsv"'
pct exec 201 -- bash -lc 'curl -s http://127.0.0.1:9093/api/v2/alerts | jq -r ".[] | [.labels.alertname,.status.state] | @tsv"'
# 手動測整條鏈路（送一則到 Gotify）
pct exec 201 -- bash -lc 'NET=$(docker inspect alertmanager-gotify-bridge -f "{{range \$k,\$v := .NetworkSettings.Networks}}{{\$k}}{{end}}"); docker run --rm --network $NET curlimages/curl:latest -s -X POST http://alertmanager-gotify-bridge:8080/gotify_webhook -H "Content-Type: application/json" -d "{\"version\":\"4\",\"status\":\"firing\",\"alerts\":[{\"status\":\"firing\",\"labels\":{\"alertname\":\"ManualTest\"},\"annotations\":{\"summary\":\"manual test\",\"description\":\"chain check\"}}]}"'

# node 清單 / OpenWrt system metrics
pct exec 201 -- python3 -c 'import json,urllib.parse,urllib.request as u; q=urllib.parse.urlencode({"query":"node_uname_info"}); data=json.load(u.urlopen("http://127.0.0.1:9090/api/v1/query?"+q, timeout=10)); [print(r["metric"].get("job"), r["metric"].get("instance"), r["metric"].get("nodename")) for r in data["data"]["result"]]'
/etc/init.d/openwrt-system-metrics status
/usr/bin/openwrt-system-metrics once
curl -s http://10.80.80.1:9105/openwrt-system-metrics.txt | sed -n '1,120p'

# Grafana dashboard 檔
pct exec 201 -- ls -lh /opt/monitoring/grafana-dashboards/
# 改 dashboard 後部署：scp -> pct push 到 grafana-dashboards/，Grafana 30s 自動 reload
#   本站總覽由 ForAI/grafana-overview-gen.py 生成：python3 grafana-overview-gen.py > homelab-overview.json 再 pct push 覆蓋

# CT270 node_exporter（原生 systemd）
pct exec 270 -- systemctl is-active node_exporter
pct exec 270 -- curl -s http://127.0.0.1:9100/metrics | grep -m1 node_uname_info
pct exec 201 -- curl -sf http://192.168.20.70:9100/metrics | grep -m1 node_uname_info   # 跨段 scrape 驗證

# Homepage（CT201 /opt/homepage，gethomepage）改 config 後：★ 必須 force-recreate 才重繪（restart 無效, .next 快取）
ssh pve24 'sudo pct exec 201 -- sh -c "cd /opt/homepage && docker compose up -d --force-recreate homepage"'
#   config 部署：本機編好 -> scp pve24:/tmp -> pct push 201 到 /opt/homepage/config/xxx.new -> 容器內 cp 覆蓋(保 inode)
# ★ ssh 遠端跑複雜命令的坑：`ssh h -- pct exec N -- bash -lc "..."` 會被 ssh 攤平失去引號；
#   要嘛用「直接 argv」(pct exec N -- cmd arg1 arg2，不經 shell)，要嘛整條包成單一參數：ssh h 'sudo pct exec N -- sh -c "..."'
```

## 10.5 CT270 Life-Ops（NocoDB）

```sh
# WAN 出網切換（OpenWrt root，助手在 /usr/local/bin；預設 BLOCKED）
ct270-net-status                 # 查目前開/關
ct270-net-on                     # 維護開網（裝套件/更新 image 前）
ct270-net-off                    # 維護後關回（務必）

# NocoDB 服務（pve24）
sudo pct exec 270 -- sh -c 'cd /opt/nocodb && docker compose ps'
sudo pct exec 270 -- sh -c 'curl -s localhost:8080/api/v1/health'   # {"message":"OK"...}

# Agent 呼叫 API（CT260；token 存 ~/.config/homelab/nocodb.env chmod600）
set -a; . ~/.config/homelab/nocodb.env; set +a
curl -s -H "xc-token: $NC_TOKEN" "$NC_URL/api/v2/meta/bases"        # 401=token 無效需重抄

# Google Calendar（MCP v2.0；憑證 ~/.config/homelab/gcal.env chmod600）
python3 ~/life-ops-mcp/get_gcal_token.py       # 一次性取/重取 refresh token（測試模式久置失效時重跑）
# 行事曆功能透過 MCP 工具用（新 claude session 載入）：
#   calendar_agenda / calendar_add / calendar_update / calendar_delete / calendar_conflicts / prep_check / daily_brief
# 註：Google API 由 CT260 發出(有 WAN)，CT270 維持斷網不受影響。

# MCP server 自我煙測（stdio 餵 JSON-RPC，不動資料可只跑 tools/list）
printf '%s\n' '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{}}' \
  '{"jsonrpc":"2.0","id":2,"method":"tools/list"}' | python3 ~/life-ops-mcp/server.py
```
