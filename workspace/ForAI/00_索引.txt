# ForAI 文件集索引（Agent 先讀我）

更新時間：2026-06-30
本資料夾是 homelab 的 AI/ops 參考文件集（由舊 ForCodex 重整而來）。
為降低維護負擔，每個敘述只放在「一個」最合適的文件，其他文件用引用而非重抄。
動手前先看本索引，找到負責該主題的文件再深入。

## 文件職責

```text
00_索引.txt                  本檔。文件集導覽 + 單一真實來源對照 + 跨檔引用規則。

家庭網路與服務拓樸文件V6.txt   【現況唯一真實來源】節點清單、VLAN/位址表、DNS records、
                             防火牆 zone/policy/DNAT、跨段 allowlist、各節點服務埠、IPv6/Tailscale 現況。
                             只描述「現在是什麼」。已併入 2026-06-29/06-30 變更。

網段指導原則.txt              【設計/規劃】各 VLAN 的用途、規劃角色、安全策略、目標網段總表
                             （含未部署的 DMZ60 / Game70 / OOB90）。談「為什麼/目標」，不列現況位址。

注意事項.txt                  【操作戒律】各子系統仍有效的 DO/DON'T + 本輪未結待辦。談「操作時小心什麼」。
                             機密檔案清單在此 §11（單一來源）。

操作規範與回滾.txt            【流程】安全改動前/中/後流程、遠端腳本修改規範（整檔推送 vs sed vs here-doc）、
                             套用前後驗證、回滾標準步驟。

監控告警.txt                  【監控單一來源】Prometheus 目標、告警規則現況、Alertmanager+Gotify、
                             CT260 Telegram notifier + DeepSeek 分層 AI、ntfy 互動推播+動作 webhook（§5g）、
                             Grafana 總覽 dashboard、必守修正原則、告警藍圖。

常用脚本.txt                  【命令】可貼上即用的命令（登入/健康檢查/PVE/防火牆/監控/DNS/備份/測試）。只放命令，背景說明在對應文件。

hl-工具箱.txt                 【工具規格】CT260 hl-* 遠端派發工具（run/mon/health/promq/push/pull/reload/snapshot）的
                             設計約束、節點對照、命令契約、擴充規則。改工具行為看這裡，非臨時命令。

備份位置及策略索引.txt        【備份】各節點標準 _backups 位置、命名規範、保留策略；機密備份只記存在。回滾步驟引用操作規範。

缺改記錄.txt                  【歷史】已修正/已刪除/已完成項目的時間線。現況不看這裡，看拓樸文件。
                             排序：無時間者按網段在前，有時間者按時間在後。

grafana-overview-gen.py       Grafana 中文總覽 dashboard 生成器（改面板改這支，重跑覆蓋 json 再 pct push）。

ha-device-inventory-gen.py    HA+OpenWrt 設備清單生成器（在 CT260 跑；抓 HA device_registry + OpenWrt leases/neigh/uci，
                              join 後輸出 HA米家設備清單.txt。已知雲端/線上 MAC 偏移在腳本 CLOUD_TO_WIRE。重跑即刷新）。
HA米家設備清單.txt            便查快照：HA 設備名/型號/雲端MAC ⋈ OpenWrt 線上MAC/IP/在線/靜態租約。由上腳本生成，勿手改。
```

## 單一真實來源對照（要查某類資訊去哪）

```text
節點 IP / 介面 / 服務埠            -> 家庭網路與服務拓樸文件V6.txt
DNS records / 命名                 -> 家庭網路與服務拓樸文件V6.txt §3
防火牆 zone / DNAT / 跨段 allowlist -> 家庭網路與服務拓樸文件V6.txt §4
VLAN 設計意圖 / 目標規劃            -> 網段指導原則.txt
操作 DO/DON'T / 待辦               -> 注意事項.txt
機密檔案清單                       -> 注意事項.txt §11
改動流程 / 遠端改檔 / 回滾          -> 操作規範與回滾.txt
Prometheus 目標 / 告警規則 / notifier / dashboard -> 監控告警.txt
可貼即用命令                       -> 常用脚本.txt
hl-* 遠端派發工具規格/節點對照     -> hl-工具箱.txt
備份檔位置 / 保留策略              -> 備份位置及策略索引.txt
某項何時改的 / 為何改               -> 缺改記錄.txt
某設備 名稱/型號/IP/MAC/靜態租約便查 -> HA米家設備清單.txt（快照；現況源仍為拓樸 §2.6 + HA registry）
```

## 跨檔引用 / 維護規則

```text
1. 同一事實只在「負責文件」維護一份，其他文件引用文件名 + 章節，不複製數字。
2. 改了現況（位址/規則/服務）：更新拓樸文件，並在缺改記錄.txt 記一筆（帶日期）。
3. 一條規則/設定被「修正或刪除」：從現況/操作文件移除，移進缺改記錄.txt。
4. 新的操作戒律放注意事項.txt；新的安全流程放操作規範與回滾.txt。
5. 機密（key/token/密碼/憑證）一律不入任何文件，只記名稱/位置/redacted。
```
